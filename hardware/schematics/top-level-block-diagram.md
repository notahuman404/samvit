## Signal Flow Summary

```
Depth frames (D435i Г—2)
        в”Ӯ
        в–ј USB3 (~11 ms: capture + transfer + SDK delivery)
Raspberry Pi 4 вҖ” depth processing (~2 ms)
        в”Ӯ
        в–ј I2C writes to PCA9685 chain (~9 ms for 144 ch update)
PCA9685 Г— 9 вҖ” PWM generation (hardware; zero CPU overhead)
        в”Ӯ
        в–ј Gate drive via 220О©
AO3400A Г— 144 вҖ” switch motor current
        в”Ӯ
        в–ј 3V / 80mA per motor (ERM rise ~15 ms)
ERM Motor Г— 144 вҖ” vibrotactile output
```

Total pipeline ~37 ms obstacleвҶ’perception; see `firmware-notes/latency-budget.md`.

## Voltage Rails Summary

| Rail | Source | Consumers | Notes |
|---|---|---|---|
| 5 V logic | PD trigger board | Pi, PCA9685 VCC, MAX98357A | Fused at 8 A |
| 3.3 V | Pi onboard LDO | PCA9685 V3V3, gate signals | Pi provides; do not load >200 mA externally |
| 3.0 V motor | Buck converter | 144 ERM motors via MOSFETs | Separate rail; keeps noise off logic |
| USB 5 V | Pi USB ports | D435i Г—2 (each 1.5 W) | Pi's USB3 ports supply cameras |
